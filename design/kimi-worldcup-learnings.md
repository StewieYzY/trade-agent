# Kimi 2026 世界杯 Multi-Agent 报告：辩论协议与校准降级机制深度分析

> 来源：Kimi 关于 2026 年世界杯赛事分析和预测报告（300+ Research Agents, 20 Dimensions）
> 分析日期：2026-07-03
> 目的：从 Kimi 的 Multi-Agent 实践中提取 **辩论协议设计** 和 **数据质量校准/降级** 两套核心机制的设计范式，为 trade-agent L3 天团提供借鉴方向

---

## 核心论点

Kimi 报告中最值得学习的不是 300 个 agent 的规模，而是两个设计哲学：

1. **辩论协议**：辩论不是"让 agent 吵架"，而是一套**分歧量化 + 分级响应 + 结构化输出**的协议。核心价值不在于达成共识，而在于让分歧本身成为可被消费的信号。
2. **校准 > 准确率**：一个准确率 67.7% 但校准良好的模型，长期优于准确率 70.2% 但未校准的模型。Agent 的输出不是"对或错"，而是"在多大程度上可靠"。

以下分两大主题展开。每个要点先给概要，再给 Kimi 原文关键段落，最后给对 trade-agent 的具体启示。

---

## 第一部分：辩论协议设计（6 个要点）

### 要点 1：分歧度作为元信号

**Kimi 的做法**：辩论不是无脑触发，也不是永远不触发。系统先量化 Agent 间的分歧度（概率估计差异的百分点数），再用这个元信号决定走哪条路径。

**关键原文**（§2.3.3）：

> 辩论协议是 Agent Swarm 架构区别于简单模型集成的核心特征。当 Agent 间分歧超过阈值时，系统自动触发辩论流程，旨在通过结构化讨论降低群体迷思风险。

> 辩论协议的底层逻辑是量化金融领域的"异质信念模型"（Heterogeneous Beliefs Model）：市场（或预测系统）中的参与者拥有不同的信息集和模型，价格（或概率）的收敛过程需要时间。通过显式建模 Agent 间的分歧，系统可以更好地量化预测不确定性，而非盲目追求虚假的共识。

**实际案例**：

> ELO 评级 Agent 预测德国战胜匈牙利的概率为 68%，战术相克 Agent 基于德国的极致高压体系将概率上调至 75%。两 Agent 分歧度为 7 个百分点（Level 1），直接加权聚合即可。但若市场分析 Agent 检测到德国队的隐含概率异常偏低（5.2% 的 Polymarket 定价 vs ELO Agent 的 11.5%），分歧度超过 50 个百分点（Level 4），则触发模型降级，暂停定量预测并等待人工审查。

**对 trade-agent 的启示**：

当前设计是固定 4 轮辩论——无论 5 个 agent 是否分歧都跑完。改进方向：Round 1 结束后计算分歧度（5 个 agent 的 conviction 标准差 + signal 一致性），用分歧度决定后续路径。不是所有分歧都值得辩论——只有分歧影响关键决策路径时才触发，否则只是浪费 Token。

### 要点 2：分级响应，而非一刀切

**Kimi 的做法**：把分歧分成四级，每个级别映射不同的处理策略。分级响应的本质是成本控制——每一次辩论都是额外的推理调用，用分级把昂贵的辩论留给真正需要它的分歧。

**关键原文**（§2.3.3 四级辩论协议）：

> Level 1: 轻度分歧（<15%）。当两个 Agent 对某事件的概率估计差异小于 15 个百分点时，采用加权平均聚合，置信度不作调整。

> Level 2: 中度分歧（15-30%）。触发情境分析辩论：分歧双方各自提供支持其判断的数据证据，第三方"仲裁 Agent"（由风险感知 Agent 兼任）评估证据质量并给出裁决。置信度降低 20%。

> Level 3: 高度分歧（30-50%）。启动多轮辩论：正方、反方、仲裁方各进行最多 3 轮陈述，每轮需提供新的数据证据。辩论结束后，若仍无法达成共识，置信度降低 50%，输出标记为"高度不确定"。

> Level 4: 极端分歧（>50%）。触发模型降级：所有涉及 Agent 的输出被标记为"冲突"，系统仅输出定性判断，暂停定量预测，等待人工审查或额外数据输入。

**对 trade-agent 的启示**：

```
我们的分歧度量化方式：
  divergence_score = conviction 标准差（5 个 agent）
  signal_consensus = signal 字段的一致性比例

分级映射（待 MVP 实测后确定阈值）：
  Level 1（低分歧）：5 agent signal 一致 + conviction std < 10
    → 跳过 Round 2-3，直接 Round 4 收敛
    → 节省 ~60% token 成本

  Level 2（中分歧）：signal 一致但 conviction std 10-25
    → 正常跑 Round 2-3
    → conviction 降低 20%

  Level 3（高分歧）：signal 不一致（3:2 或 2:2:1）
    → 正常跑 Round 2-3
    → 输出 signal: "conflict"
    → conviction 降低 50%

  Level 4（极端分歧）：signal 完全分散（1:1:1:1:1 或类似）
    → 跳过 Round 2-3
    → 直接输出 signal: "conflict" + 分歧报告
    → 标记"需人工审查"
```

### 要点 3：每轮强制"新数据证据"，防止辩论退化

**Kimi 的做法**：多轮辩论最容易退化成"双方各执一词反复复读"。约束是：每一轮陈述必须提供新的数据证据，不能只是换个说法重复上一轮的观点。

**关键原文**（§2.3.3 Level 3）：

> Level 3: 高度分歧（30-50%）。启动多轮辩论：正方、反方、仲裁方各进行最多 3 轮陈述，**每轮需提供新的数据证据**。

**对 trade-agent 的启示**：

辩论的价值在于引入新信息打破僵局。如果一轮辩论没有新证据产出，就该终止并标记"不可解决"。

改进方向：在 Round 2 的 prompt 中增加约束——
```
"你在 Round 2 的回应中，必须引用至少一个 Round 1 中未被讨论的数据维度。
如果所有相关数据已在 Round 1 中被引用，请明确声明'无新增数据支撑'，
系统将提前终止辩论并标记为'信息穷尽'。"
```

同时在 `AgentOutput` 中增加字段：
```json
{
  "new_evidence": ["本轮新引用的数据点"],
  "evidence_exhausted": false  // 是否已穷尽所有可用数据
}
```

当 5 个 agent 中 ≥3 个标记 `evidence_exhausted: true` 时，Round 3 自动跳过。

### 要点 4：第三方仲裁 Agent，而非分歧双方自己决胜负

**Kimi 的做法**：仲裁由风险感知 Agent 兼任——它本身不是分歧的当事方，职责是评估双方的证据质量并给出裁决。让分歧双方自己"投票决胜负"是有风险的，因为多数派未必正确。

**关键原文**（§2.3.2 战略层角色）：

> | 层级 | Agent 角色 | 核心职责 | 关键 Prompt 约束 |
> |------|-----------|---------|-----------------|
> | 战略层 | 风险感知 Agent | 监控模型漂移、数据异常、黑天鹅事件 | "自动降级触发条件" |

**关键原文**（§2.3.3 Level 2）：

> Level 2: 中度分歧（15-30%）。触发情境分析辩论：分歧双方各自提供支持其判断的数据证据，**第三方"仲裁 Agent"（由风险感知 Agent 兼任）评估证据质量并给出裁决**。

**对 trade-agent 的启示**：

我们的 Devil's Advocate 本质上就是这个角色，但当前设计让它在 Round 3 挑刺后，Round 4 由 synthesizer agent 综合输出。改进方向：

- Devil's Advocate 的职责从"挑刺"升级为"仲裁"——不只找漏洞，还要**评估各 agent 的证据质量**
- Round 4 的 synthesizer 不再"综合所有观点"，而是**基于 Devil's Advocate 的仲裁报告**做出最终判断
- Devil's Advocate 的输出增加：
  ```json
  {
    "evidence_quality_assessment": {
      "buffett": "strong_data_support",
      "munger": "moderate_data_support",
      "feng_liu": "weak_data_support"
    },
    "recommendation": "defer_to_buffett_munger_consensus"
  }
  ```

### 要点 5：产出是"分歧报告"，不是"谁说服谁"

**Kimi 的做法**：辩论的终点不是达成共识，而是输出一份结构化的分歧报告——量化分歧幅度、记录各方主张与证据、调整置信度、给出消费建议。"未解决"本身就是一个有价值的输出。

**关键原文**（§2.3.4 合成算法）：

> Agent 辩论的最终产物是 20 个模型的概率输出。合成算法负责将这些异质输出融合为统一的概率估计。

> 当模型间分歧超过阈值时，合成结果自动扩大置信区间并降低置信度等级。

**关键原文**（§2.3.3 底层逻辑）：

> 通过显式建模 Agent 间的分歧，系统可以更好地量化预测不确定性，而非盲目追求虚假的共识。

**对 trade-agent 的启示**：

这是整套设计里最反直觉、也最重要的。强行让分歧收敛成单一答案，会丢失不确定性信息；保留分歧、量化分歧，才是诚实的。

当前 Round 4 输出的是"综合结论 + 保留的分歧点"。改进方向——Round 4 的输出从"共识结论"变为**结构化分歧报告**：

```json
{
  "divergence_report": {
    "divergence_level": "level_2",
    "divergence_score": 18.5,
    "signal_distribution": {"bullish": 3, "neutral": 1, "bearish": 1},
    "key_disagreements": [
      {
        "topic": "ROE 下降趋势的解读",
        "bull_case": {"agent": "duan", "evidence": "行业周期性因素", "strength": "moderate"},
        "bear_case": {"agent": "feng_liu", "evidence": "结构性恶化信号", "strength": "strong"}
      }
    ],
    "confidence_adjustment": -0.2,
    "final_signal": "bullish",
    "final_conviction_range": [55, 70],
    "recommendation": "关注 ROE 趋势是否在下季度企稳"
  }
}
```

### 要点 6：分歧需要时间收敛

**Kimi 的做法**：系统中的参与者拥有不同的信息集和模型，概率的收敛是一个需要时间的过程。系统不该急于在第一轮就追求共识——有些分歧会随着新数据注入自然收敛。

**关键原文**（§2.3.3 底层逻辑）：

> 市场（或预测系统）中的参与者拥有不同的信息集和模型，价格（或概率）的收敛过程需要时间。

**关键原文**（§7.4 模型漂移报告）：

> 模型漂移（Model Drift）是指模型性能随时间推移而系统性恶化的现象。在长达 39 天的世界杯赛事中，漂移是不可避免的——球队的真实实力在演化，而模型的参数估计基于的是滞后数据。

> 每日摘要将输出：Calibration drift（预测概率 vs 实际频率的校准偏移）>5% 时触发权重重校准。

**对 trade-agent 的启示**：

辩论协议要和时间维度配合——允许悬而未决，让持续校准机制在新数据到来后自然收敛分歧。

改进方向：L4 监控层增加**分歧追踪**功能——
- 如果 L3 辩论产出了 `signal: "conflict"`，L4 在下一次数据更新时自动重跑 L3
- 对比两次辩论的分歧度变化：如果分歧收敛（新 divergence_score < 旧的 50%），说明新数据解决了分歧
- 如果分歧持续或扩大，标记为"结构性分歧"，提醒人工介入

## 第二部分：校准与降级机制（4 个要点）

### 要点 1：校准 > 准确率

**Kimi 的做法**：这是最需要扭转的认知。一个准确率 67.7% 但校准良好的模型，长期优于准确率 70.2% 但未校准的模型——因为后者在高置信预测上系统性犯错，实际 ROI 为负。

**关键原文**（§2.3.4 概率校准）：

> 概率校准是合成后处理的关键步骤。未经校准的模型输出常常存在系统性的概率偏差——例如，当模型输出 60% 胜率时，实际胜率可能只有 52%。

> 校准质量的评估指标是 Brier Score 及其分解：Reliability 衡量预测概率与实际频率的一致性（越低越好），Resolution 衡量预测区分不同结果的能力（越高越好），Uncertainty 反映结果本身的随机性（不可控）。

> 值得注意的是，校准与准确率并非同一概念。SportBot AI 的分析表明，一个校准优化的模型（准确率 67.7%，Brier Score 0.197）在长期来看优于一个未校准的高准确率模型（准确率 70.2%，Brier Score 0.341）——因为后者在高置信预测上系统性犯错，导致实际 ROI 为负。

> 这一发现对本研究的启示是：**宁可降低点估计的"惊艳度"，也要确保概率校准的诚实性。**

**对 trade-agent 的启示**：

我们的 `conviction: 0-100` 是 agent 主观给的，从未校准过。Phase 3 积累足够推荐历史后，用 Brier Score 回溯——conviction=80 的股票后来有多少只确实涨了？如果实际胜率只有 55%，说明 agent 系统性高估了 conviction。

校准结果反馈到两处：
1. prompt 调整：告诉 agent "你的 conviction 系统性偏高 25 点，请校准"
2. 后处理映射：conviction_raw=80 → conviction_calibrated=55

### 要点 2：降级条件必须可量化、可自动执行

**Kimi 的做法**：降级触发条件全是精确阈值，不需要人工判断。为什么？因为人工判断会成为瓶颈——当数据质量恶化时，系统可能已经在"自信地犯错"，而人工审查的响应速度跟不上。

**关键原文**（§2.1.2 数据质量控制 + §2.3.4 模型降级条件）：

> 当关键变量缺失率超过 30% 时，自动触发模型降级机制。

> 模型降级条件在以下情形自动触发，无需人工干预：
> - 近 10 场预测 RPS 恶化超过 20%
> - 输入数据源延迟超过 24 小时
> - 关键变量缺失率超过 30%

> 在以下情形触发人工审查：
> - 多模型分歧超过 50%
> - 预测结果与主流学术模型方向相反
> - 极端事件（如种子队 0-3 输球）后的首批预测

**关键原文**（§2.3.2 风险感知 Agent）：

> | Agent 角色 | 核心职责 | 关键 Prompt 约束 |
> |-----------|---------|-----------------|
> | 风险感知 Agent | 监控模型漂移、数据异常、黑天鹅事件 | "自动降级触发条件" |

**对 trade-agent 的启示**：

我们的 L2/L3 没有降级机制——数据不好也照常输出。改进方向：定义精确的降级触发条件——

```python
# L2 Scout 降级条件
L2_DEGRADATION_TRIGGERS = {
    "key_fields_missing": {
        "fields": ["pe_ttm", "pb", "roe_latest", "cash_flow_operating"],
        "threshold": 2,  # 缺失 ≥2 个关键字段时降级
    },
    "f_score_incomplete": {
        "threshold": 6,  # F-Score 只能算 ≤6/9 项时降级
    },
    "data_staleness": {
        "financials_age_days": 180,  # 财报数据超过 180 天
    },
}

# L3 Council 降级条件
L3_DEGRADATION_TRIGGERS = {
    "agent_error_rate": {
        "threshold": 0.4,  # ≥40% agent 输出 error/timeout 时降级
    },
    "l2_data_quality_degraded": True,  # L2 已降级时 L3 自动降级
}

# 降级行为
DEGRADATION_ACTIONS = {
    "l2_degraded": {
        "confidence_cap": 50,           # confidence 上限 50
        "verdict_override": "watch",    # 强制 watch
        "label": "data_quality_degraded"
    },
    "l3_degraded": {
        "skip_rounds": [2, 3],          # 跳过辩论，只做 Round 1 + 4
        "confidence_cap": 40,
        "label": "council_degraded"
    },
}
```

### 要点 3：三层不确定性分解

**Kimi 的做法**：把不确定性拆成三层——参数不确定性（样本不足导致，可改善）、模型不确定性（建模假设差异，可管理）、结构性不确定性（固有随机性，不可消除）。这个分解的价值不在分解本身，而在于它防止你掉进优化陷阱。

**关键原文**（§2.4.1 三层不确定性分类）：

> 本研究将预测不确定性系统性地分解为三个层次，每层对应不同的量化方法与表述规范。

> 第二层：模型不确定性（Model Uncertainty）。源于模型选择的主观性。为什么用 Dixon-Coles 而非 Bivariate Poisson？不同的建模选择可能导致显著不同的预测结果。本研究通过模型集合（Ensemble）的方式处理模型不确定性：20 个模型的输出离散度即为模型不确定性的度量。

> 第三层：结构性不确定性（Structural Uncertainty）。源于足球赛事的固有随机性——这是任何模型都无法消除的不确定性层次。红牌、点球误判、球员突发伤病、门柱折射等低概率高影响事件，属于结构性不确定性的范畴。

> 实践中，三层不确定性的相对贡献随预测对象而异。以夺冠概率为例：
> - 参数不确定性贡献了总方差的约 25%
> - 模型不确定性贡献了约 35%
> - 结构性不确定性贡献了约 40%，是最大且不可消除的成分

**对 trade-agent 的启示**：

这个分解防止你掉进优化陷阱——如果你把 40% 的结构性不确定性误当成"模型不够好"，就会永无止境地调参加特征，但天花板就在那里。

对应到 trade-agent：

| 不确定性层次 | 在 trade-agent 中的表现 | 能否改善 |
|-------------|----------------------|---------|
| **参数不确定性** | 数据不完整（财报缺失字段、估值分位计算样本不足） | ✅ 通过更多数据源/更长时间窗口改善 |
| **模型不确定性** | L2 prompt 歧义、L3 agent 投资哲学差异导致的不同判断 | ✅ 通过 prompt 校准/案例库收敛 |
| **结构性不确定性** | A 股政策突变、黑天鹅事件、管理层个人行为 | ❌ 不可消除，只能接受并标注 |

改进方向：在 L3 输出的分歧报告中，标注分歧来源——
```json
{
  "divergence_source": {
    "parameter": "low",       // 数据完整，不是数据问题
    "model": "high",          // agent 投资哲学差异导致
    "structural": "medium"    // 存在不可预测的外部因素
  }
}
```

如果分歧主要来自 `structural`，再怎么辩论也没用——直接标注"不可解决"。

### 要点 4：优雅降级——数据不可得时切简化模型，而非全盘挂起

**Kimi 的做法**：当部分数据缺失（比如缺乏事件级数据），系统自动降级为纯结果模型继续运行，而不是报错停止。这保证系统在不同数据可用性条件下都能产出"力所能及"的结论。

**关键原文**（§2.2.3 xG 增强层）：

> 当数据不可得时（部分历史比赛缺乏事件数据），自动降级为纯结果模型（仅使用进球数据）。这种"优雅降级"（Graceful Degradation）设计确保了模型在不同数据可用性条件下的稳健运作。

**关键原文**（§2.1.2 数据可用性四问）：

> 3. 样本量（Sample Size）：近 3 年国家队数据是否 ≥15 场/队？俱乐部数据是否 ≥20 场？不满足则标注"低样本"并触发降级处理。

**关键原文**（§2.3.2 Prompt 约束）：

> 以"战术相克 Agent"为例，其 Prompt 强调："你的分析必须基于历史数据和统计模式，而非主观印象。对于任何'风格相克'的论断，必须提供至少 20 场同类 matchup 的历史胜率作为支撑。**无法找到足够样本时，明确标注'统计不显著'。**"

**对 trade-agent 的启示**：

当数据不全时，系统应该继续运行但降低输出精度，而非报错退出。

改进方向——定义 L2/L3 的降级运行模式：

```
正常模式（full_data）：
  - L2：完整 200 tokens 特征输入 → 正常 verdict + confidence
  - L3：5+1 agent 全天团辩论

降级模式（degraded_data）：
  - L2：精简特征输入（仅有基础+财报） → confidence 上限 50，verdict 强制 watch
  - L3：只做 Round 1（各自表态）→ 跳过辩论 → 输出分歧报告

极简模式（minimal_data）：
  - L2：跳过（数据不足以支撑 LLM 判断）
  - L3：跳过
  - 输出：仅 L1 量化排序结果 + 标注"数据不足，未做 LLM 分析"
```

关键原则：**系统永远有输出，只是输出的精度和置信度随数据质量而变。**

## 第五部分：Kimi 方法论架构 vs trade-agent 对应关系

作为上下文补充，Kimi 报告的方法论是一个五层金字塔：

```
Layer 5（校准层）：四层回测框架、概率校准、置信区间构建、自动降级机制
Layer 4（协作层）：~300 个 Agent 的三层架构，结构化辩论协议处理模型分歧
Layer 3（模拟层）：≥100,000 次蒙特卡洛迭代，三情景平行模拟
Layer 2（模型层）：20 个异质模型覆盖评级、进球分布、过程指标、ML 和情境因子
Layer 1（数据层）：7 个层级的数据融合架构，每条数据须通过"可用性四问"
```

trade-agent 对应结构：

```
Layer 5 → L4 监控 + 案例库校准 + 分歧追踪（Phase 3）
Layer 4 → L3 天团辩论（5+1 agent × 分级辩论协议）
Layer 3 → 不适用（不做蒙特卡洛）
Layer 2 → L1 量化筛选 + L2 LLM 初筛
Layer 1 → 数据采集层（22 维 + 三层漏斗 + 数据完整性检查）
```

## 第六部分：f2-debate-protocol-fix 落地状态（2026-07-10）

f2 change 将上述借鉴点落地到 trade-agent L3，状态如下：

### 完全借鉴（2 项）
- **辩论要点 1（分歧度作为元信号）→ D1**：R1 后 `compute_divergence` 算 signal_consensus + conviction_std → level 分流。低/极高分歧跳 R2/R3 省 token。
- **辩论要点 5（产出是分歧报告）→ D4**：SynthesizerOutput 加 6 个增量字段（divergence_level/key_disagreements/confidence_adjustment 等），叠加不替换 final_verdict。

### 部分借鉴（7 项）
- **辩论要点 2（分级响应）→ D1 改造**：Kimi 的 15%/30%/50% 阈值是世界杯概率场景调的，trade-agent 用保守默认（signal_consensus 0.8/0.6）+ 标注待 MVP 校准。
- **辩论要点 3（每轮新数据证据）→ D2 降级**：原硬约束降为 soft warning——实证 L3 输入仅 21 量化字段，R2 无新维度可引，硬约束触发编造-校验死循环。字段保留作 f3 enabling carrier。
- **辩论要点 4（第三方仲裁）→ D3 改造**：DA 加事实回查（回查 features 实际值）而非 LLM 评 LLM 文字游戏。已知局限：features 仅 21 字段，DA 想回查的定性维度没有。
- **辩论要点 6（分歧时间收敛）→ D9 暂缓**：L4 分歧追踪依赖全市场跑通 + watchlist 稳定，当前仅预留 schema 字段。
- **校准要点 1（校准 > 准确率）→ D8 只标注**：watchlist 标 `calibration_status: "uncalibrated"`，Brier 实际计算 Phase 3（conviction 主观分非概率，概念别扭）。
- **校准要点 2（降级条件可量化）→ D5/D6 分场景**：L2 优雅降级（critical 齐但 financials 不齐 → watch+cap50）、L3 fail-fast（financials_floor 入口）+ L3 运行时降级（error rate≥0.4）。
- **校准要点 3（三层不确定性）→ D7 只 structural**：parameter vs model 细分 LLM 自评不靠谱，只标 structural（high 时标「不可解决」省死辩论）。

### 不借鉴（1 项）
- **校准要点 4（优雅降级，永远有输出）**：L3 单只深研若数据不足硬出结论 = 600900 复读茅台悲剧，L3 用 fail-fast 更诚实。L2 批处理则走优雅降级。

### 未解决的根因（f3 范畴）
f2 优化辩论**过程**而非辩论**信息基础**——L3 输入仍 21 量化字段，无定性维度（主营构成/研报/治理/竞品）。真正让「深」变深需 f3-l3-research-dossier 升级输入为结构化研究档案，届时 D2 soft warning 升回 hard gate、D3 事实回查能覆盖定性维度。详见 `design/kimi-worldcup-learnings.md` 配套偏移分析。
