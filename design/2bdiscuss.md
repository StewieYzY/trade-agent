## 话题列表

### 1. L1 因子权重 (50/30/20) 的校准 🔴

**问题**：目前质量 50% / 估值 30% / 安全边际 20% 是拍脑袋还是有理据？需要回测验证吗？

**为什么重要**：权重决定了最终 200 只候选的排序，直接影响 L2 的输入质量。

**涉及模块**：L1 Screener

---

### 2. 冯柳 agent 的决策框架 🔴 🔒

**问题**：段永平和芒格的 prompt 已写了草稿，但冯柳的「弱势研究法」是天团里最难蒸馏的一个——他本质上是做「别人不看的我看」，这怎么 prompt 化？

**为什么重要**：冯柳是 A 股独有的 alpha 来源，如果蒸馏不到位，他的 agent 会和段永平/巴菲特同质化，失去存在价值。

**涉及模块**：L3 Analyst Council

---

### 3. 辩论中 agent 之间的信息不对称 🟡

**已覆盖**：design-v1 6.5 节的 `debate.py` 骨架已定义信息可见性控制：
- Round 1：agent 彼此隔离（防从众）
- Round 2：每个 agent 看到其他 4 人的论点（促质疑）
- Round 3：Devil's Advocate 看到全部讨论（找盲区）

**未覆盖**：上下文窗口会不会爆炸？Round 2 给每个 agent 看其他 4 人的**全文**还是**摘要**？6 个 agent × 4 轮，token 消耗估算？

**涉及模块**：L3 Analyst Council

---

### 4. L4 催化事件检测的具体设计 🔴

**问题**：`detect_catalyst()` 目前是一个黑箱函数名。催化事件的数据源是什么？怎么判断「催化」vs「噪音」？

**涉及模块**：L4 Monitor

---

### 5. 回测与验证体系 🟡

**已覆盖**：`prd-rule&case.md` 覆盖了历史案例库的校准用法（发现盲区、验证逻辑一致性）。

**未覆盖**：系统性回测——跑历史数据验证 L1 因子有效性、L2 筛选准确率、L3 判断一致性。是否需要？如果需要，怎么做？

**涉及模块**：全局

### 6. 系统交互形态 ✅

**已解决**：Docker + Streamlit，写入 design-v1「九、技术决策」。

**决定**：MVP 用 Streamlit（纯 Python 数据看板），后续如需复杂交互可迁移到 FastAPI + React。

### 7. L2 输出的质量保证 🔴

**问题**：LLM 会不会把同一只股票今天判 deep_dive 明天判 skip？temperature=0 够不够？需要多轮投票吗？

**涉及模块**：L2 Scout Agent
