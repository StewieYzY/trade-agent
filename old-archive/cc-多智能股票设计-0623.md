# cc-多智能股票设计-0623

> 基于 UZI-Skill 仓库改造 · 面向 A 股价值投资小白的选股 + 多 agent 研判系统设计思路

---

## 一、第一性原理：拆开「快」与「深」的伪冲突

### 用户的两个核心需求

- **需求 A（快）**：快速选股，在几千只 A 股里找到低估值/低 PE/甚至低热点的票，在合适时点布局。
- **需求 B（深）**：多 agent 分析师天团（巴菲特/芒格/段永平蒸馏而成），通过辩论达成共识，给已有仓位做下一步决策。

### 伪冲突的本质

直觉把它们放在了**同一条流水线上**——好像「天团辩论」是「选股」的下一个环节，天团太重所以选股不可能快。这是把两件事搞混了。

```
        ❌ 你以为的流程（错）
  选股(快) ──→ 天团辩论(慢) ──→ 决策
      把天团塞进选股环节，快不起来

        ✅ 真正的结构
  全市场筛子(快 · 机械)         仓位研判(慢 · 主观)
        ↓                            ↓
  产出一个 watchlist            天团辩论这一只
  (几十只候选)                 (已经选过的标的)
```

**核心结论**：快和深不是同一条管线的两个环节，是**两条独立管线，用 watchlist 接口连接**。

- **选股阶段根本不需要 agent，更不需要辩论。** 选股是**排序问题**，不是判断问题。5000 只票用机械规则打分排序，留下 30-50 只候选。这步巴菲特本人来了也是用规则筛（他早年就是翻 Moody's 手册找 PE<15、ROE>15% 的票）。
- **天团辩论的价值，在于对「已经筛出来的少数标的」做深度研判。** 它要回答的不是「这只票便宜吗」（规则能答），而是「巴菲特会不会真买这个生意」「段永平觉得它的商业模式对不对」「芒格会不会觉得这是太难的生意」——这是质性判断，才需要蒸馏的语气库和辩论。

UZI-Skill 现有的全部资产——22 维采集、三级容错、Wave 并发、resume、stock_features 特征层、pipeline 三段式——都是为**深管线**服务的。要新加的，是一个**轻量的快管线**，然后复用深管线的能力。

## 二、戳破一个「离谱」设想

用户说「找低估值、低 PE、甚至**低热点**的股票」想布局入场。前两个（低估值/低 PE）是价值投资正道。但**「低热点 + 等合适时点入场」要拉一把**——这不是价值投资，是择时/动量派的话术，价值投资小白恰恰死在这一步最多。

第一性原理：

- 「低热点」**不创造价值**。一只无人问津的票，可能因为它真被低估（机会），也可能因为它就该被遗忘（**价值陷阱**——夕阳行业、财务造假、治理崩坏）。**低热度本身不区分这两者。** 纯粹找冷门股，会收到一堆价值陷阱。
- 「合适时点入场」**是择时**。而价值投资的整个立论基础就是「**不择时**」——格雷厄姆说市场先生是来伺候你的不是来指导你的，巴菲特说「如果你不愿意持有一只股票十年，那就十分钟也不要持有」。小白试图「等一个低点入场」，90% 会变成「涨了追高、跌了割肉」。

**修正后的需求**：你真正想要的，应该是**「在冷门/低估值区间里，用财务质量筛掉价值陷阱，等到出现基本面催化或估值修复信号时纳入 watchlist」**。这样「低热点」才有意义——它是一个**筛选维度**（排除高热度=高被炒风险），不是「等一个入场时点」的择时信号。

价值投资里真正的「时点」概念是：**催化剂**（财报超预期、行业政策、管理层换届、分红提升），而不是 K 线低点。这个区分要内化为选股系统的第一性原理。

## 三、改造思路：双管线 + 共享地基

```
┌─────────────────────────────────────────────────────────────────────┐
│                      共享地基 (从 UZI 复用)                            │
│   22 维采集 · 三级容错 · Wave 并发 · resume · stock_features 特征层    │
│   (这些全是深管线资产 · 已确认要保留 · 一行不改)                       │
└─────────────────────────────────────────────────────────────────────┘
            ▲                                    ▲
            │ 复用 fetcher + features             │ 复用 fetcher + features + 天团
            │                                    │
┌───────────┴──────────────┐        ┌───────────┴──────────────────────┐
│  快管线 · 选股 (Screener)   │        │  深管线 · 研判 (Panel)            │
│                            │        │                                  │
│  · 全市场快照 1 次拉取       │        │  · 对 1 只票跑 22 维深采           │
│  · 机械规则打分排序         │ 产出    │  · stock_features 派生 60 特征     │
│  · 输出 watchlist (30-50只) │───────→│  · 天团 agent 辩论 (3-5 轮)        │
│                            │ watchlist│  · consensus + 买卖区间          │
│  目标: 30 秒 - 2 分钟       │        │  · 这就是 UZI 现有的 deep-analysis │
│  不跑 agent · 不辩论        │        │  目标: 5-10 分钟                   │
└────────────────────────────┘        └──────────────────────────────────┘
            │
            │ 定期跑 (每周/每月)
            │ watchlist 落盘，增量更新
            ▼
      持仓/关注清单管理
```

好处：**只新写一条快管线，深管线几乎原样保留**（甚至可砍掉不需要的部分，比如游资 24 评委、Serenity AI 卡位——价值投资，这些流派价值不大）。

### 3.1 快管线（要新建的——改造的主体）

现在**没有**、但最需要的部分。设计成**纯机械、可复现、30 秒出结果**的筛子。

#### 选股的「三道漏斗」

```
全市场 ~5000 只
   │
   ▼ 第一道：硬性剔除 (硬约束 · 一票否决)
   ├─ 剔 ST/*ST/退市风险
   ├─ 剔上市<3年 (无足够财务历史 · 格雷厄姆要求)
   ├─ 剔市值过小 (<50亿 · 流动性+抗风险)
   ├─ 剔金融/券商 (估值指标对金融股失效 · 巴菲特式价值股不适用)
   ├─ 剔周期股 (可选 · 看你能不能承受 · 钢铁/煤炭/航运/化工)
   └─ 剔高质押 (实控人质押>70%)
   │
   ▼ 第二道：价值质量双因子 (软排序 · 不剔除只打分)
   ├─ 估值因子: PE_TTM < 行业中位 ×0.7  +  PB < 2  +  股息率 > 2%
   ├─ 质量因子: F-Score ≥ 7  +  ROE 5年>15%  +  现金流为正 3年
   ├─ 安全边际: (DCF内在价 - 当前价)/当前价 > 30%
   └─ 综合分 = 加权 (质量 50% + 估值 30% + 安全边际 20%)
   │
   ▼ 第三道：低热度过滤 (特殊需求 · 但用对方式)
   ├─ 换手率分位 < 30% (不活跃 · 排除被炒的)
   ├─ 雪球/东财热度排名靠后 (低关注度)
   └─ 近60日涨幅 < 20% (排除刚炒完的 · 避免接盘)
   │
   ▼ watchlist: 30-50 只候选 · 按 "质量分×安全边际" 排序
```

**为什么是这三道漏斗**：

- **第一道硬剔除**：把「不该看的」一次清掉。价值陷阱高发区——ST、次新、小盘、周期、治理有问题的。纯排除法，不涉及判断。
- **第二道双因子**：价值选股核心。直接用 **Piotroski F-Score + 格雷厄姆数 + 一个简化的 DCF 安全边际**，三个都是经过 30 年学术验证的公式，不要自己发明。
- **第三道低热度**：个人需求保留，但**放在最后**——「冷门」只有在「又便宜又高质量」的前提下才有意义。先质量后冷门，不会捡到价值陷阱。

#### 全用 akshare，且 UZI 已有现成接口

| 选股需求 | akshare 接口 | UZI 是否已用 |
|---|---|---|
| 全市场快照（PE/PB/市值/换手率） | `stock_zh_a_spot_em` | ✓（data_sources.py） |
| 估值分位 | `stock_a_pe_and_pb` | ✓（fetch_valuation） |
| 财务三表（算 F-Score/ROE） | `stock_financial_abstract` | ✓（fetch_financials） |
| 股息率 | `stock_individual_spot_xq` | ✓（雪球 fallback） |
| 同行对标 | `stock_zh_valuation_comparison_em` | 部分（fetch_peers） |
| 行业分类（剔周期/金融） | `stock_board_industry_cons_em` | ✓（fetch_industry） |

**不需要重写任何数据层**——UZI 的 22 维采集把单只票数据全抓回来了，要做的只是**新增一个「全市场批量快筛」入口**，复用现有 fetcher 对候选池逐只拉特征。这是复用的最大杠杆点。

### 3.2 深管线（天团）—— UZI 的资产，但有取舍

UZI 现有的 `deep-analysis` 就是深管线，但**现在的 66 评委是为「通用分析」设计的，对价值投资小白来说，三分之二用不上**。要做减法：

#### 砍掉的流派（对价值投资噪音 > 信号）

- **F 组 24 个游资**：全部砍。价值投资，龙虎榜/涨停/题材是噪声不是信号，且只看 A 股。
- **H 组科技领袖 4 个**：可砍。「买龙头」思维，和价值投资「买低估」方向不同。
- **I 组 Serenity**：可砍。AI 卡位是题材/趋势派逻辑。
- **D 组技术 4 个**：可砍。Stage 2/均线是趋势择时，和价值投资冲突。

#### 保留并强化的流派（天团核心）

围绕 **4 个原型**重做天团，而不是 66 个：

```
天团 (建议 5-8 个 agent · 不要 66 个)
─────────────────────────────────────────
① 价值守护者 (巴菲特/芒格蒸馏)
   · 关注 ROE 护城河 FCF · 拒绝看不懂的生意
   · 蒸馏源: 致股东信 / 芒格 Daily Journal 演讲

② 安全边际猎手 (格雷厄姆/卡拉曼蒸馏)
   · 7 项硬指标 + 30% 折扣红线
   · 蒸馏源: 《证券分析》/《安全边际》

③ 成长质量检验 (费雪/林奇蒸馏)
   · PEG / 闲聊法 / 十倍股特征
   · 蒸馏源: 《怎样选择成长股》/《彼得林奇的成功投资》

④ 中国价投 (段永平/张磊蒸馏)  ← 用户点名要的
   · 做对的事 / 商业模式 / 长期持有
   · 蒸馏源: 段永平博客问答 / 张磊《价值》

⑤ 质疑者/魔鬼代言人 (独立角色 · 不对应真人)
   · 专门挑刺: 价值陷阱? 周期顶? 财务造假?
   · 这是辩论能收敛的关键 · 没有它天团会一致看好
```

**为什么砍到 5-8 个，而不是保留 51 个**：辩论框架的数学规律是——**agent 越多，越难收敛，共识越稀释成「平均值」**。5 个有明确立场的 agent 辩论 3 轮，比 51 个跑规则的可复现得多。目标是「小白要一个能帮决策的判断」，不是「模拟一个完整的 66 人投委会」。

#### 蒸馏语录的正确方式

UZI 现在的 `investor_personas.py` 是**查台词模板表**——给个 signal，从该评委的签名句列表随机抽一句填占位符。这对「快速出报告」够了，但**对「多 agent 辩论」远远不够**。真正的辩论需要 agent **能针对具体论据回应**，不是念台词。

蒸馏一个 agent 的正确层级（从浅到深）：

```
Level 1 (UZI 现状): 台词模板表 → 查表填空 · 不能辩论
   └─ person: {buffett: {bullish: ["好生意...", ...]}}

Level 2 (推荐起点): 规则 + 风格系统提示 → LLM 在风格内自由生成 · 可辩论
   └─ system: "你是巴菲特。你的决策框架: ROE>15%, 护城河,
              不懂的生意一律 skip。你的说话风格: 平实、举例、
              会引用『市场先生』。现在评价这只票..." + 特征数据

Level 3 (进阶): 知识库 RAG + 立场一致性约束
   └─ system + 真实语录向量库 (致股东信) +
      "你的判断必须能用语录库里的原话佐证"
```

**强烈建议从 Level 2 起步，不要一上来做 Level 3**。Level 2 用一个 system prompt（内嵌该投资人决策框架 + 风格）+ 喂入 stock_features 的特征数据，就能让 LLM 在该投资人立场内生成论点、回应其他 agent。这才是「辩论」能发生的基础。Level 3 的 RAG 是后期优化，小白阶段没必要。

### 3.3 两条管线的连接点：watchlist

整个设计的关键接口。快管线产出 watchlist，深管线消费它：

```json
watchlist.json (快管线产出 · 深管线入口)
{
  "generated_at": "2026-06-23",
  "candidates": [
    {
      "ticker": "002273.SZ",
      "name": "水晶光电",
      "score": 87,
      "f_score": 8,
      "pe_ttm": 18.2,
      "pb": 1.8,
      "roe_5y_avg": 17.3,
      "dividend_yield": 3.1,
      "safety_margin_pct": 35,
      "heat_rank": 0.28,
      "flags": ["low_heat", "high_dividend"],
      "rationale": "F8 / ROE 17% / 股息 3.1% / 估值分位 28%"
    }
  ]
}
```

这个 watchlist 就是**日常打开看的东西**。每周跑一次快管线刷新，新进/跌出做 diff，**只有想深入研究的某一只，才丢给深管线天团**。

### 4.3 最关键的两个工程判断

**判断 1：快管线要不要用 agent？不要。**
选股是排序，不是判断。F-Score、格雷厄姆数都是确定的数学公式，LLM 在这步只会添乱（幻觉、不一致）。快管线应该是**纯 Python 函数**，可复现、可单测、30 秒跑完。这一步 UZI 的 `investor_criteria.py` 那套 `Rule(check=lambda f: ...)` 模式正好能复用——把 F-Score 的 9 条写成 9 个 Rule 就行。

**判断 2：辩论框架用 LangGraph 还是简化版？小白建议简化版。**
LangGraph 强大但重（状态图、reducer、条件边），对「5 个 agent 辩论 3 轮」是过度工程。建议**先用最简单的串行对话**实现：

```
Round 1: 5 个 agent 各自独立给初判 (并行 · 5 个独立 LLM 调用)
Round 2: 每个 agent 看到其他 4 人的论点 → 修订自己的立场 (并行)
Round 3: 「质疑者」agent 综合挑刺 → 各 agent 最终表态
收敛: 投票 + 标记分歧点 (分歧本身是信息 · UZI 的设计)
```

用纯 Python + 一个 `for round in range(3)` 循环 + 共享 state dict 就能写，比上 LangGraph 简单一个数量级。等跑顺了、想加「动态轮数/收敛检测/人在回路」，再迁移到 LangGraph 也不迟。**不要在第一天就上框架。**

## 五、对「离谱」需求的最终回应

开头说这个需求「有点离谱」。判断是：**不是离谱，是被表述混了。** 拆开之后：

- 「快速选股 + 低估值低 PE」→ 完全合理，是价值投资正道，且 akshare + UZI 的资产能直接支撑，30 秒一个 watchlist。
- 「低热点」→ 合理，但要作为**排除维度**（剔除被炒的），不是「等冷门转热」的择时信号。
- 「合适时点布局入场」→ 唯一建议修正的。价值投资的「时点」是**催化剂**（财报/分红/政策），不是 K 线低点。把这个词从「择时」重新定义为「**等待基本面催化**」，系统才不会把小白带进择时的坑。
- 「多 agent 天团辩论」→ 合理且有价值，但它是**深管线的活**，对已选定的标的用，不塞进选股环节。

**唯一真正的冲突，是把快和深放在了同一条流水线上。** 拆成两条管线、用 watchlist 连接，冲突就消失了。UZI 的遗留产物里，深管线那套（采集/特征/渲染）几乎可以原样复用，主要新增一个快管线 + 重做一个精简的天团。

3. **探讨「催化剂」怎么建模**——把「合适时点」落到一个可监控的事件系统（财报日历/分红公告/政策），替代 K 线择时。
4. **继续探讨**——对上面判断有不服想辩的（比如「真的就需要一点择时」或「天团还是想要 51 个」），往深里聊。

### Piotroski F-Score 九项标准（速查）

**盈利能力 (Profitability)**

1. ROA——当年 ROA 为正 +1 分
2. 经营性现金流——当年为正 +1 分
3. ROA 变化——当年 ROA > 上年 ROA +1 分
4. 应计项目——(经营现金流/总资产) > (ROA/总资产) +1 分

**杠杆、流动性与资金来源 (Leverage, Liquidity and Source of Funds)**

5. 杠杆变化——长期杠杆率 < 上年 +1 分
6. 流动比率变化——流动比率 > 上年 +1 分
7. 股本变化——当年未发行新股 +1 分

**经营效率 (Operating Efficiency)**

8. 毛利率变化——毛利率 > 上年 +1 分
9. 资产周转率变化——资产周转率 > 上年 +1 分

**评分**：0-9 分。8-9 分为强，0-2 分为弱。注意不同行业平均分有差异，跨行业比较需谨慎。
